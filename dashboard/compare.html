The finding

It isn't length — it's whether the instruction can be followed or must be understood. AfterQuery's approved tasks describe a symptom or a real-world failure and make the agent reason out the fix (persona bug reports, terse tickets, or planted defects with no map). Our first drafts read like a solved Jira ticket — they hand over the API contract and a step list, so a mid-tier model just types it in. Note AfterQuery's ask-10 is also 4 KB yet approved: it's long but describes constraints and symptoms (corrupt file kills the batch, retries double-bill), not a checklist.
